# TLELESS_BlockChain_Horizon1
=======


---

Blockchain Development Environment Setup

This repository contains the configuration and setup for running a local Ethereum development environment using Hardhat and connecting it to MetaMask for transaction interaction and testing.

The setup allows developers to simulate blockchain transactions locally with pre-funded accounts and test smart contract interactions without using a public blockchain network.

---

Running Local Blockchain

A local Ethereum node is started using Hardhat.

npx hardhat node

This launches a JSON-RPC server at:

http://127.0.0.1:8545

The node automatically generates 20 development accounts with test ETH.

Example output:

Account #0: 0xf39fd6e51aad88f6f4ce6ab8827279cfffb92266
Balance: 10000 ETH

These accounts can be used for testing transactions locally.


---

Connecting MetaMask to Local Network

MetaMask was configured to connect to the Hardhat local blockchain.

Network Configuration

Field	Value

Network Name	Hardhat Local
RPC URL	http://127.0.0.1:8545
Chain ID	31337
Currency Symbol	ETH


After saving the network configuration, MetaMask connects directly to the local blockchain node.


---

Importing Hardhat Account

To access the pre-funded test ETH, a Hardhat account private key was imported into MetaMask.

Steps:

1. Open MetaMask


2. Select Import Account


3. Paste a private key from the Hardhat node output



Example:

Private Key:
0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80

After importing the account, the wallet displays:

Balance: 10000 ETH
Network: Hardhat Local


---

Transaction Interaction

Once connected, MetaMask can interact with the local blockchain.

Actions supported:

Action	Description

Account creation	Multiple accounts available
Transaction signing	MetaMask signs transactions
Balance verification	Test ETH available
Blockchain queries	JSON-RPC requests handled by Hardhat


The Hardhat terminal logs blockchain interactions such as:

eth_chainId
eth_getBalance
eth_blockNumber

These logs confirm successful communication between MetaMask and the local blockchain.

Notes

The ETH generated by Hardhat is for development only.

These private keys are publicly known and must never be used on live networks.

The local blockchain resets whenever the Hardhat node is restarted.
