Blockchain Development Environment Setup

This repository contains the configuration and setup for running a local Ethereum development environment using Hardhat and connecting it to MetaMask for transaction interaction and testing.

The setup allows developers to simulate blockchain transactions locally with pre-funded accounts and test smart contract interactions without using a public blockchain network.

Running Local Blockchain

A local Ethereum node is started using Hardhat.

npx hardhat node

This launches a JSON-RPC server at:

http://127.0.0.1:8545

The node automatically generates 20 development accounts with test ETH.

Example output:

Account #0: 0xf39fd6e51aad88f6f4ce6ab8827279cfffb92266
Balance: 10000 ETH

These accounts can be used for testing transactions locally.


---

Connecting MetaMask to Local Network

MetaMask was configured to connect to the Hardhat local blockchain.

Network Configuration

Field	Value

Network Name	Hardhat Local
RPC URL	http://127.0.0.1:8545
Chain ID	31337
Currency Symbol	ETH


After saving the network configuration, MetaMask connects directly to the local blockchain node.


---

Importing Hardhat Account

To access the pre-funded test ETH, a Hardhat account private key was imported into MetaMask.

Steps:

1. Open MetaMask


2. Select Import Account


3. Paste a private key from the Hardhat node output



Example:

Private Key:
0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80

After importing the account, the wallet displays:

Balance: 10000 ETH
Network: Hardhat Local


---

Transaction Interaction

Once connected, MetaMask can interact with the local blockchain.

Actions supported:

Action	Description

Account creation	Multiple accounts available
Transaction signing	MetaMask signs transactions
Balance verification	Test ETH available
Blockchain queries	JSON-RPC requests handled by Hardhat


The Hardhat terminal logs blockchain interactions such as:

eth_chainId
eth_getBalance
eth_blockNumber

These logs confirm successful communication between MetaMask and the local blockchain.

Local Ethereum Development & Transaction Environment

This project sets up a complete local Ethereum blockchain environment for testing transactions and smart contract interactions.
The system runs a private blockchain node locally and connects it with a wallet interface to simulate real blockchain transactions.

The repository demonstrates how a developer can create a local blockchain network, generate funded accounts, connect a wallet, and execute transactions without using a public blockchain.

What We Implemented

The following development environment and workflow were successfully implemented:

1. Local Blockchain Network

A private Ethereum blockchain was launched using Hardhat.

Command used:

npx hardhat node

This starts a local JSON-RPC blockchain server running at:

http://127.0.0.1:8545

The node automatically generates 20 development accounts with test ETH.

Example output:

Account #0
Balance: 10000 ETH

These accounts allow testing transactions without real cryptocurrency.


---

2. Wallet Integration

The local blockchain was connected to MetaMask to enable wallet-based interaction with the network.

Network configuration:

Parameter	Value

Network Name	Hardhat Local
RPC URL	http://127.0.0.1:8545
Chain ID	31337
Currency Symbol	ETH


After adding this network, MetaMask connects directly to the local blockchain node.


---

3. Importing Blockchain Accounts

To access the pre-funded development accounts, one of the Hardhat private keys was imported into MetaMask.

Example:

Private Key:
0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80

Once imported, the wallet displays:

Balance: 10000 ETH
Network: Hardhat Local

This confirms the wallet is connected to the local blockchain.


---

4. Transaction Interaction

With the wallet connected, blockchain operations can be simulated.

Supported interactions include:

Operation	Description

Account management	Multiple blockchain accounts available
Balance verification	Each account funded with test ETH
Transaction signing	MetaMask signs blockchain transactions
Network queries	Hardhat processes JSON-RPC requests


When transactions or wallet queries occur, the Hardhat node logs activity such as:

eth_chainId
eth_getBalance
eth_blockNumber

These logs confirm successful communication between the wallet and the blockchain node.
